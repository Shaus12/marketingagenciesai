# How to Judge Quality

This file defines what "good enough to publish" means. Every output goes through this filter.

**Model:** Use Opus 4.6 for quality judgment. Never let Sonnet or Haiku decide if something is ready.

---

## The Core Principle

**If it doesn't make you feel something, don't publish it.**

"Feeling something" means:
- Surprise ("I didn't know that")
- Recognition ("That's exactly my situation")
- Discomfort ("I don't want to admit that's true")
- Curiosity ("I need to know what happens next")
- Admiration ("That's a smart way to think about it")

"Fine" is not a feeling. "Interesting" is not a feeling. If the best word you can use is "nice" — kill it.

---

## Quality Tiers

### Tier 1: Publish immediately
- Makes you stop and re-read it
- You'd screenshot it and send it to someone
- It contains a specific insight nobody else is saying
- It sounds unmistakably like the founder — not like generic AI content

### Tier 2: Refine and publish
- The insight is there but the wording is soft
- Needs one more pass to sharpen the language
- Could be more specific (add a number, a name, a real example)
- Almost there — 15 minutes of editing makes it Tier 1

### Tier 3: Use the insight, rewrite the execution
- The underlying idea is good but the expression is generic
- Sounds like it could have come from any AI creator
- Needs to be completely rewritten with the founder's specific experience
- The "what" is right but the "how it's said" is wrong

### Tier 4: Kill it
- Generic advice anyone could give
- No tension, no surprise, no edge
- Sounds like AI wrote it (symmetrical structure, buzzwords, lists without insight)
- Publishing this would dilute the brand — better to post nothing

**Rule: Only Tier 1 and Tier 2 get published. Tier 3 gets reworked. Tier 4 gets deleted.**

## The 5 Quality Checks

Run every output through these in order:

### 1. The Screenshot Test
"Would someone screenshot this and send it to a builder friend?"
- If yes → passes
- If "maybe" → needs sharpening
- If no → kill it

### 2. The Attribution Test
"If you removed the name, could you tell the founder wrote this?"
- If yes → passes (it has his voice, specificity, and directness)
- If it could be anyone → fails (too generic)

### 3. The Scroll Test
"If this appeared in your feed between a meme and a friend's photo, would you stop?"
- If yes → passes
- If you'd scroll past → the hook is too weak

### 4. The "So What" Test
"After reading this, does the audience know something new or feel something different?"
- If yes → passes
- If they think "okay, and?" → it's not providing enough value or tension

### 5. The Cringe Test
"Would the founder be embarrassed if a respected peer saw this?"
- If he'd be proud → passes
- If he'd think "that's a bit try-hard" or "that's generic" → rework it
- This catches: hype language, fake urgency, overpromising, guru energy

## Common Quality Failures

### The AI Voice Problem
AI-generated content has tells:
- Perfect parallel structure ("Not just X, but Y. Not just A, but B.")
- Smooth transitions ("Moreover," "Furthermore," "Additionally")
- Balanced lists (always exactly 3 or 5 items)
- Hedging ("It's important to note that...")
- Superlatives without evidence ("The most powerful way to...")

**Fix:** After generating, ask: "Would a human text this to a friend?" Rewrite until it sounds like a text message, not an essay.

### The "Good Advice" Problem
Content that is technically correct but completely generic:
- "Validate your idea before building" — true but boring
- "Focus on solving a real problem" — everyone says this
- "Build an audience first" — generic creator advice

**Fix:** Make it specific to the founder's experience. Not "validate first" but "I built 25 apps without validating. 22 of them made zero dollars. Here's what the other 3 had in common."

### The "Too Polished" Problem
Content that feels too clean, too structured, too perfect:
- Perfect grammar throughout
- Smooth narrative arc
- No rough edges or personality

**Fix:** Add imperfection. Real thoughts don't flow perfectly. Sometimes a sentence is just two words. Sometimes you start with "Look." or "Here's the thing." The rawness IS the authenticity.

---

## When In Doubt

Ask one question: **"Is this something the founder would say unprompted in a conversation with a builder friend over coffee?"**

If yes — publish.
If no — either it's not his voice, not specific enough, or not insightful enough. Fix whatever is off.
